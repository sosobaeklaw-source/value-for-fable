# VFF 벤치 — 결과 (2026-06-14)

세 줄 요약:
1. **압축 강제는 품질 부채였다.** 그걸 들어낸 v2가 원래 v1보다 중립 채점표에서 **+10.9점**(76.2 → 87.1) 높다.
2. **Sonnet+v2는 Opus와 노이즈 안에서 동률이다** — 두 차례 중립 재채점에서 v2 ≈ Opus(약 95–101%). 단 기본 진단·분량글에선 v2가 Opus와 같거나 앞서고, 깊은 추론(아키텍처 결정·복잡 성능진단)에선 Opus가 5–7점 앞선다.
3. **가성비**: v2는 Opus 출력의 약 0.30배 비용(70% 저렴) → 품질당 비용효율 약 3배. "전부 Opus급"은 아니지만 가성비는 분명히 선다.

원래 2026-06-11 벤치는 원자료가 안 남아 재현 불가였다. 이 디렉토리는 재현 가능한 하네스 + 원자료다. 심판은 전부 Claude(Opus) 블라인드이며 한계는 맨 아래.

---

## Phase 1 — 원래 "VFF 5/6승" 미재현

공정 베이스라인(일반 좋은 프롬프트)·분량/설명형 과제를 추가해 재현 시도. 4조건 × 6과제(주관 3·객관 3) × 블라인드 Claude 심판. 원자료 `raw.json`. **풀 v1(압축 포함)이 주관 평균 꼴찌, 1위 0회.** 원인은 압축 편향이 길이 제약(H2: 350자에 282자=20% 미달, 거의 객관적 위반)과 설명형에서 역효과. 객관 3과제는 4조건 전부 통과(변별 안 됨, ceiling).

## Phase 2 — ablation: 압축이 부채

VFF를 4암으로 갈라 비교(`ablation_phase2b_raw.json`): 풀 v1 / VFF−압축 / 진단구조만 / 맨 Sonnet. **평균 순위(1=최고): VFF−압축 1.67 · 진단구조만 2.00 · 맨 Sonnet 3.00 · 풀 v1 3.33.** 진단 구조는 자산(VFF계열이 맨 Sonnet 이김), 압축은 순 부채(압축 뺀 암 1위, 압축 센 v1 꼴찌). VFF를 최악→최선으로 바꾼 건 압축 제거.

부수 발견: 풀 v1을 위임 서브에이전트에 통째 주입하면 글쓰기 과제에서 도구 호출 무한루프(331회)에 빠짐. "도구 호출 금지, 텍스트만" 한 줄로 해소.

## Phase 3 — 채점표 편향 적발

압축 뺀 v2(`output-styles/vff-v2.md`)와 복합진단을 더한 v3를 처음 평가했을 때 v2/v3가 Opus와 동급 내지 우세로 보였으나, **그 채점표가 v3 프롬프트 문구를 그대로 베껴 특정 진단 스타일을 보상**하고 있었다(coached한 답이 채점표대로 이기는 순환). 이 결과는 폐기하고 중립 채점으로 다시 했다.

## Phase 4 — 중립 재채점 (이게 최종)

중립 채점표(특정 스타일 선호 금지) + 독립 심판 2명(시니어 엔지니어·질문 당사자 관점) 평균. VFF는 Sonnet용이므로 Opus+VFF 통제군 제외, 맨 Opus를 현실 레퍼런스로. 새 생성 없이 기존 답변 재채점. 원자료 `v2_raw.json`·`v3_raw.json`, 점수 `rejudge_scores.json`·`rejudge_v1v2opus_scores.json`.

**중립 재채점 A — v1·v2·Opus, 과제 H1/H2/H3/H4 (0–100 평균)**

| 과제 | Sonnet+v1 | Sonnet+v2 | 맨 Opus |
|---|---|---|---|
| H1 진단 | 84.0 | **90.5** | 84.5 |
| H2 350자글 | 62.0 | **83.0** | 74.5 |
| H3 RAG설명 | 80.0 | 88.0 | **93.0** |
| H4 성능진단 | 79.0 | 87.0 | **93.0** |
| **평균** | 76.2 | **87.1** | 86.2 |

**중립 재채점 B — v2·v3·Opus, 과제 H1/H4/H5/H6 (0–100 평균)**

| 과제 | Sonnet+v2 | Sonnet+v3 | 맨 Opus |
|---|---|---|---|
| H1 진단 | 86.5 | 88.5 | 85.5 |
| H4 성능진단 | 84.5 | 76.5 | **91.0** |
| H5 아키텍처결정 | 86.0 | 80.0 | **92.0** |
| H6 회귀진단 | 82.0 | 88.5 | **89.0** |
| **평균** | 84.8 | 83.4 | **89.4** |

### 무엇이 확정됐나

- **v1 → v2 = +10.9점**(76.2 → 87.1, 재채점 A). 이게 "올라온 v1 대비 v2 업그레이드"의 깨끗한 중립 수치. v1은 Opus의 88%, v2는 약 95–101%.
- **v3는 v2를 못 이긴다**(재채점 B: v3 83.4 < v2 84.8). 복합진단 분해·예시는 중립 기준 효과 없음. **v3 폐기, v2가 keeper.**
- **v2 ≈ Opus(노이즈 안 동률).** 두 재채점이 엇갈린다 — A에선 v2 87.1 ≈ Opus 86.2(101%), B에선 v2 84.8 < Opus 89.4(95%). 차이는 과제 구성(B엔 어려운 추론 H5·H6이 많음). 두 런의 약 5점 차이가 곧 노이즈 바닥이다.
- **과제 성격으로 갈린다**: v2가 Opus와 같거나 앞서는 곳 = 기본 진단(H1), 분량글(H2, v2 83 vs Opus 74.5 — Opus가 분량 초과로 감점). Opus가 앞서는 곳 = 설명 완성도(H3 +5), 깊은 추론(H4·H5·H6 +5–7).

## 가성비 — 정직한 형태

- **품질**: 두 중립 런 종합 v2 약 86 vs Opus 약 88 = Opus의 **약 95–100%**. 일반 과제 믹스에선 노이즈 안 동률, 가장 어려운 추론만 Opus 실제 우위.
- **비용**: v2 평균 출력은 Opus의 약 절반 길이 + Sonnet 출력 단가가 쌈($15 vs $25) → 출력 비용 **약 0.30배(70% 저렴)**. 입력 단가까지 더하면 절감 더 큼.
- **종합**: 품질당 비용효율 **약 3배**. 정형 작업 다수를 Sonnet+v2로 처리하고 소수의 어려운 추론(아키텍처·복잡 성능진단)만 Opus로 라우팅하면 절감이 크다.

## 검증된 것 / 안 된 것 / 반증된 것

- **검증됨**: v1→v2(압축 제거)는 견고한 +10.9점 개선(ablation + 중립 재채점). v2 ≈ Opus(노이즈 안), 약 30% 비용.
- **검증 안 됨**: v2→v3(복합진단 분해)는 중립 기준 효과 없음. 제품은 v2.
- **반증됨**: 원래 "VFF 5/6승·257점", "1/4 압축", 그리고 한때 보였던 "v3가 Opus 압승"은 모두 표본/채점표 편향 산물.

## 한계 (과신 금지)

- **심판이 Claude(Opus) 단일 가족**. 단, 그 편향은 경쟁자 Opus에 유리한 방향이라 Sonnet 점수를 부풀리진 않는다. 사람 골드 심판 미수집(강인성 선택).
- **표본 작음**: 과제당 1–2회·심판 2명. 두 중립 런이 약 5점 흔들린다 = 노이즈 바닥. 방향(v2≫v1, v2≈Opus)은 견고하나 소수점은 못 믿는다. 조이려면 과제·반복을 늘려야 함.
- **과제 범위**: 진단·조언·결정·짧은글 중심. 순수 창작·장문 서술 미검증.
