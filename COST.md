# 비용·토큰 분석 — Sonnet+VFF vs Opus

"VFF로 아낀 토큰이 Opus를 안 쓸 만큼의 가성비가 되는가, 아니면 비슷해서 그냥 Opus 쓰는 게 나은가." 이 문서는 그 질문에 단가 기준 실수치로 답한다.

> **2026-06-14 갱신**: 아래 본문의 "약 1/4 압축" 같은 초기 수치는 단판 관측치였다. 공정 재검증([bench/RESULTS.md](bench/RESULTS.md))의 실측값은 — VFF v2 출력은 Opus의 약 절반 길이, 출력 비용 기준 **약 0.30배(70% 저렴)**, 품질은 Opus의 약 95–100%(노이즈 안 동률, 깊은 추론만 Opus 우위). 1/4(0.25)은 길이비가 가장 좋았던 한 과제의 값이고, 평균 길이비는 약 0.5다.

## 먼저 — 토큰 개수와 토큰당 단가는 다른 얘기다

같은 작업을 시키면 **소비 토큰 개수 자체는 모델을 바꿔도 거의 안 변한다.** Opus든 Sonnet이든 같은 파일을 읽고 같은 길이로 답하면 토큰 수는 비슷하다. 그래서 "Sonnet이 토큰을 덜 먹어서 싸다"는 건 틀린 그림이다. 가성비의 레버는 두 개로 나뉜다.

1. **토큰당 단가가 다르다** — 이건 작업과 무관하게 확정된 차이다.
2. **VFF가 출력 토큰을 줄인다** — 실측 평균 출력 길이는 v2가 Opus의 약 0.5배(bench/RESULTS.md). 단가까지 곱하면 출력 비용 약 0.30배. 작업 종류에 따라 편차가 크다.

## 단가 (2026-06, 100만 토큰당)

| 모델 | 입력 | 출력 | Sonnet 대비 |
|---|---|---|---|
| Sonnet 4.6 | $3 | $15 | 기준 |
| Opus 4.8 | $5 | $25 | Opus가 1.67배 (=Sonnet이 40% 쌈) |
| Fable 5 | $10 | $50 | Fable이 3.33배 |

정직하게 짚을 단서 하나: 예전 Opus는 Sonnet의 **5배**($15/$75)였는데, Opus 4.8이 $5/$25로 내려오면서 격차가 **1.67배**로 좁혀졌다. 즉 "그냥 Opus 쓰자"의 비용 페널티가 과거보다 훨씬 작아졌다. VFF의 비용 명분은 1년 전보다 약하다 — 그래도 최소 40%는 확정 차이다.

## 작업 종류가 절감폭을 가른다

같은 토큰 개수라면 Sonnet은 무조건 Opus의 60%(40% 절감)다. 여기에 VFF의 출력 압축이 얹히는데, 그게 얼마나 먹히냐는 **입력/출력 비율**에 달렸다.

### 입력이 많은 작업 — 에이전트 코딩

컨텍스트·파일·툴 결과를 매 턴 다시 먹이는 코딩 세션은 입력이 비용을 지배한다. 입력 200만 + 출력 20만 토큰 가정:

| | 입력 | 출력 | 합 | Opus 대비 |
|---|---|---|---|---|
| Opus 4.8 | $10.0 | $5.0 | **$15.0** | 100% |
| 맨 Sonnet | $6.0 | $3.0 | **$9.0** | 60% |
| Sonnet+VFF (출력 절반) | $6.0 | $1.8 | **$7.8** | 52% |

여기선 입력이 지배해서 VFF의 출력 압축은 $9 → $7.8 정도만 더 깎는다. **절감의 대부분은 VFF가 아니라 그냥 단가 40% 차이에서 나온다.** 코드 출력은 산문처럼 1/4로 줄지 않는다(절반 가정).

### 출력이 많은 작업 — 글쓰기·긴 분석·답변

출력이 비용을 지배하면 VFF의 압축이 제대로 먹힌다. 입력 5만 + 출력 40만 토큰 가정:

| | 입력 | 출력 | 합 | Opus 대비 |
|---|---|---|---|---|
| Opus 4.8 | $0.25 | $10.0 | **$10.25** | 100% |
| Sonnet+v2 (출력 약 0.5배) | $0.15 | $3.0 | **$3.15** | 31% |

출력형에서 Sonnet+v2는 출력 비용 기준 Opus의 **약 0.30배(약 1/3)**까지 내려간다(실측, bench/RESULTS.md). 다만 압축률은 작업 종류 편차가 크고, 더 짧은 게 항상 더 나은 건 아니다 — 분량 지정 과제는 오히려 그 값에 정확히 맞춰야 한다(v1의 압축 강제가 미달로 감점된 게 그 사례).

## 라우팅 기준

비용은 "비슷"하지 않다 — 최소 40%, 출력형이면 약 1/3까지 벌어진다. 진짜 변수는 품질이고, 판단은 작업 종류로 가른다.

- **구조가 명확한 작업(정형 코딩·글·리서치)**: Sonnet+VFF가 Opus 품질에 근접하면서 비용 절반 이하 → 쓸 가치 있음.
- **낯선 도메인·다층 진단·아키텍처 설계 같은 깊은 추론**: Opus의 생추론이 이기고 VFF는 지식 천장을 못 메운다 → 그냥 Opus.
- **타협안**: Sonnet 초안 + Opus 리뷰 1패스. 풀 Opus보다 싸면서 Sonnet의 깊이 구멍을 막는다. 단 호출 2회라 단순 작업엔 오히려 손해.

## 한 줄 결론

"비슷해서 무의미"는 **비용 측면에선 틀렸고**(최소 40%, 출력형이면 더), **품질 측면에선 부분적으로 맞다** — 깊은 추론 과제에선 VFF로 Opus를 대체 못 한다. 그래서 정답은 "둘 중 하나"가 아니라 **작업 종류로 라우팅**하는 것이다.
